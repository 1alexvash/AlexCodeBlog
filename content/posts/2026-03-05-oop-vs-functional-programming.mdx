### OOP way

If you would create your game with the object-oriented programming way, your app and submarine would roughly look something like this.

```javascript
const submarine = {
  move(direction) {
    // Moving
  },

  attack(target) {
    // Attacking
  },

  upgrade(system) {
    // Upgrading
  }
};

```

And then in the OOP way you will be able to run and command with your submarine like this

```javascript
submarine.move("UP");
submarine.upgrade("Shiled_Level_2")
submarine.upgrade("Speed_Level_3")
```

### Functional programming way

In a purely functional way, without any inspiration from the object style your functionality would like something as follows:

```javascript
function move(direction) {
    // Moving
}

function attack(target) {
  // Attacking
}

function upgrade(system) {
  // Upgrading
}

```

Very simple, right?

## Bottom line

As you can see the first approach is based around objects as the core unit of organizing the code while the second around functions, but in  essence, both of those approaches are doing the same thing, they run your code and help you organize your program, and in the real-world application, you are in fact probably using both of those approaches at the same time.

It's just that modern frameworks like React they lean more heavily towards the functional programming way, while other frameworks like Angular or NestJS lean towards the OOP way.

In essence, in the OOP way, you send `O(f)` while in the functional way `f(O)`
